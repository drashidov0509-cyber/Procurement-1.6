# 📥 Как получить готовый Procurement.exe за 10 минут

> Эта инструкция написана для тех, кто никогда не программировал.
> Вам не нужно устанавливать ни Python, ни какие-либо программы на свой компьютер.
> GitHub соберёт `.exe` за вас бесплатно на своих серверах.

---

## ⏱ Что вы получите в итоге

Один файл `Procurement.exe` (~25 МБ) который:
- ✅ Запускается двойным кликом
- ✅ Не требует установки
- ✅ Работает на любом Windows 10/11
- ✅ Можно копировать на флешку

---

## 🎯 ШАГ 1. Регистрация на GitHub (3 минуты)

GitHub — это бесплатный сайт где хранится код. Регистрация как в Gmail.

1. Откройте 👉 **https://github.com/signup**
2. Введите ваш email → **Continue**
3. Придумайте пароль → **Continue**
4. Придумайте имя пользователя (латиницей, например: `ivan-petrov`) → **Continue**
5. Подтвердите что вы не робот (картинка) → **Create account**
6. На почту придёт код — введите его на сайте
7. Дальше можно пропустить все опросы (кнопка **Skip personalization** внизу)

✅ Готово, у вас есть GitHub-аккаунт.

---

## 🎯 ШАГ 2. Создать репозиторий (1 минута)

Репозиторий — это «папка» с кодом на GitHub.

1. После входа нажмите зелёную кнопку **«Create repository»** на главной странице
   (или сверху справа на «+» → **New repository**)
2. Заполните:
   - **Repository name**: `procurement` (можно любое название)
   - **Public** — оставить как есть (Public)
   - **НЕ ставьте** галочки «Add a README file» и другие
3. Нажмите зелёную **«Create repository»**

✅ Появится страница с инструкциями — мы будем загружать файлы.

---

## 🎯 ШАГ 3. Загрузить файлы программы (2 минуты)

1. На странице нового репозитория найдите строчку:
   > _"...or upload an existing file"_ — это синяя ссылка, нажмите на неё.
   (Если её нет — кликните по кнопке **Add file** → **Upload files**)

2. **Распакуйте** архив `procurement-desktop.zip` на компьютере
   (правый клик → «Извлечь всё»)

3. Откройте полученную папку `desktop_app`
4. **Выделите ВСЕ файлы и папки внутри** (Ctrl+A)
   В ней должно быть:
   - `app.py`
   - `api.py`
   - `scrapers.py`
   - `classifier.py`
   - `excel_export.py`
   - `ui.html`
   - `requirements.txt`
   - `build.py`
   - `README.md`
   - папка `.github` ⬅ **ОЧЕНЬ ВАЖНО, эта папка должна быть!**

5. **Перетащите** все выделенные файлы мышкой в окно GitHub
   (на белую область где написано «Drag files here»)

6. Подождите 30 секунд пока всё загрузится

7. Внизу страницы зелёная кнопка **«Commit changes»** → нажмите

✅ Файлы залиты на GitHub.

---

## 🎯 ШАГ 4. Запустить сборку (1 минута)

GitHub автоматически сам начнёт собирать `.exe`. Проверим:

1. В вашем репозитории сверху есть вкладки: **Code · Issues · Pull requests · Actions**...
2. Кликните на **«Actions»**
3. Увидите запущенную сборку с названием **«Build Windows EXE»**
   - Жёлтый кружочек 🟡 = идёт сборка
   - Зелёная галочка ✅ = успешно собралось
   - Красный крестик ❌ = ошибка (значит файлы загружены неправильно)
4. Подождите **~5 минут** пока кружочек не станет зелёной галочкой

> 💡 Если ничего не происходит — обновите страницу через минуту.

---

## 🎯 ШАГ 5. Скачать готовый Procurement.exe (1 минута)

1. На странице Actions кликните на завершённую сборку (зелёная галочка)
2. Прокрутите вниз — увидите блок **«Artifacts»**
3. Там будет файл **«Procurement-Windows»** — нажмите на него
4. Скачается ZIP-архив
5. **Распакуйте** его — внутри ваш `Procurement.exe`!

✅ **Готово!** Двойной клик по `.exe` — программа запускается.

## ⚠️ Если что-то пошло не так

### Сборка с красным крестиком ❌

Откройте страницу сборки → найдите шаг где ошибка (стрелочка вниз раскроет лог).

**Чаще всего проблема — забыли загрузить папку `.github`:**
- Вернитесь в репозиторий
- Проверьте есть ли там папка `.github` в списке файлов
- Если нет: **Add file → Upload files** и загрузите её отдельно

### Windows ругается «Защитник Windows заблокировал этот файл»

Это нормально для неподписанных программ:
1. Нажмите **«Подробнее»**
2. Нажмите **«Выполнить в любом случае»**

Если очень мешает — можно купить сертификат подписи кода (~$80/год), но для личного использования не нужен.

### Антивирус удаляет .exe

Это **ложное срабатывание** — PyInstaller (который собирает .exe) часто срабатывает на антивирусах. Добавьте файл в исключения антивируса.

### Сборка идёт уже 15 минут

Может GitHub перегружен. Откройте Actions → нажмите на сборку → справа сверху **«Re-run all jobs»**.

## 🔄 Как обновить программу позже

Если я добавлю новые функции (новые страны, источники и т.д.):
1. Зайдите в свой репозиторий на GitHub
2. Кликните на файл который изменился (например `scrapers.py`)
3. Нажмите карандашик ✏️ (Edit)
4. Замените всё содержимое на новое (Ctrl+A → Delete → вставить новое)
5. Внизу — **Commit changes**
6. GitHub сам пересоберёт .exe — скачаете обновлённый
